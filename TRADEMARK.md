# Trademark Policy

This trademark policy governs the use of the "CMAS" name, logos, and branding in connection with the CMAS (Claude Multi-Account Switcher) open source project.

## Trademarks

The following are trademarks of Hoa TQ:

- **CMAS** (word mark)
- **Claude Multi-Account Switcher** (word mark)
- The CMAS logo and icon

## What You Can Do

### Use the Code Freely

The CMAS source code is licensed under CC BY-NC-SA 4.0. You are free to:

- Use, modify, and share the code for **non-commercial purposes**
- Create derivative works (non-commercial only)
- Learn from and reference the code

### Make Factual Statements

You may make accurate, factual statements about your relationship to the project:

- "Based on CMAS"
- "Built with CMAS technology"
- "Fork of CMAS"

### Contribute to the Project

Contributors to the official CMAS repository may use the trademarks when discussing their contributions.

## What You Cannot Do

### Use CMAS Branding for Forks

If you create a fork or derivative work, you **must**:

- Choose a different name that does not include "CMAS" or "Claude Multi-Account Switcher"
- Remove or replace all CMAS logos and icons
- Update the bundle identifier (e.g., `com.boykioyb.cmas`) to your own
- Make it clear that your fork is not the official CMAS

### Imply Official Endorsement

You may not:

- Use "CMAS" or "Claude Multi-Account Switcher" as your product name
- Use the CMAS logo as your application icon
- Suggest that your fork is the official version
- Imply that Hoa TQ endorses your product

### Create Confusion

You may not use the trademarks in any way that:

- Suggests your product is created by or affiliated with Hoa TQ
- Could cause confusion between your product and the official CMAS
- Disparages the CMAS project or its author

## Branding Locations

For those creating forks, the following files contain branding that should be updated:

| File | Contains |
|------|----------|
| `src-tauri/tauri.conf.json` | Product name, bundle ID |
| `src-tauri/icons/` | Application icons |
| `public/favicon.png` | Web favicon |
| `src/assets/hero.png` | Logo image |

## Examples

### Acceptable

- "MyTool - based on CMAS"
- "This project is a fork of CMAS"
- "Inspired by CMAS"

### Not Acceptable

- "CMAS Pro"
- "CMAS for Linux"
- "Better CMAS"
- Using the CMAS logo for your fork
